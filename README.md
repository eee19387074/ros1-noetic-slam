# ROS 1 Noetic SLAM with a 2-Wheeled Differential Drive Robot

This repository demonstrates how to perform **Simultaneous Localization and Mapping (SLAM)** using **ROS 1 Noetic** on a **2-wheeled differential drive robot**. It supports both **simulation (Gazebo)** and **real hardware** setups and is intended for learning, research, and prototyping purposes.

### Frame Descriptions

* **map**: Global reference frame generated by the SLAM algorithm
* **odom**: Local odometry frame published by Gazebo / wheel encoders
* **base_link**: Robot base reference frame
* **left_wheel / right_wheel**: Driven wheel frames (published by Gazebo)
* **chassis**: Main robot body frame
* **caster_wheel**: Passive support wheel

## ğŸš€ Running the Simulation

### 1. Launch Gazebo Simulation

```bash
roslaunch gazebo bringup.launch
```

This starts:

* Gazebo world
* Differential drive robot
* Controllers and sensors

---

### 2. Start SLAM

#### Option A: GMapping

```bash
roslaunch slam gmapping.launch
```

#### Option B: Hector SLAM

```bash
roslaunch slam hector.launch
```

#### Option C: Cartographer

```bash
roslaunch slam cartographer.launch
```

---

### 3. Teleoperate the Robot

```bash
rosrun teleop_twist_keyboard teleop_twist_keyboard.py
```

Move the robot around to generate the map.

---

### 4. Visualize in RViz

```bash
roslaunch rviz display.launch
```

RViz displays:

* Laser scan
* TF tree
* Occupancy grid map
* Robot model
